# Twin MVP Scope

This document defines the minimal system required for the first
working Twin platform.

---

# 1. Core Runtime

Required:

- session loop
- artifact generation
- critique
- evaluation signals
- creative state updates
- memory storage

---

# 2. Governance

Required:

- approval state
- intervention rules
- change records

---

# 3. Product Surfaces

Required:

Private Studio  
Staging Habitat  
Public Habitat

---

# 4. Source Library

Basic ingestion of:

text  
images  
references

---

# 5. Storage

Required:

database (metadata)  
artifact storage  
session history

---

# 6. Not Required for MVP

Visitor analytics  
complex UI environments  
distributed runtime  
automated training pipelines

---

# 7. MVP Goal

The goal of the MVP is to prove that the Twin can:

- run creative sessions
- generate artifacts
- evaluate them
- preserve memory
- allow human review
- publish selected outputs
